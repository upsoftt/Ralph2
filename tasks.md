# Задачи: Ralph 2.0 — Audit Fixes

## Спринт 1: Критические баги и race conditions
**[Role: Backend Architect]**

- [ ] {{TASK:1.1}} Исправить _save_launch_state для работы с массивом процессов
  **ПОДРОБНОСТИ:**
  - **Что сделать:** В `ralph-tracker-web.py` функция `_save_launch_state()` вызывает `proc.poll()` на значении из `LAUNCHED_PROCESSES`, но после рефакторинга значение может быть списком `[Popen, Popen]`, а не одним `Popen`. Вызов `list.poll()` бросает `AttributeError`, который молча проглатывается `except: pass`.
  - **Как сделать:** Обновить `_save_launch_state()` — если значение `isinstance(v, list)`, итерировать по элементам списка, для каждого проверять `proc.poll() is None` и сохранять массив PID. Обновить `_restore_launch_state()` аналогично — восстанавливать как список `_PidHandle`. Также проверить `update_status()` где `launch_running` уже обработан, но могут быть другие ссылки.
  - **Ограничения:** Не ломать обратную совместимость с уже сохранёнными `launch_state.json` (может содержать старый формат с одним PID).
  - **Критерии приёмки:** Запустить проект с multi-step launch.json (два shell-шага). Убедиться что `launch_state.json` корректно сохраняет оба PID. Перезапустить веб-сервер — оба процесса отображаются как запущенные.

- [ ] {{TASK:1.2}} Исправить heartbeat чтобы не перезаписывал paused флаг
  **ПОДРОБНОСТИ:**
  - **Что сделать:** В `ralph-overseer.js` heartbeat `setInterval` (каждые 5 сек) читает `status.json`, обновляет `heartbeat` и записывает обратно. Но если веб-сервер между чтением и записью установил `paused: true`, heartbeat перезапишет его на `paused: false`.
  - **Как сделать:** В heartbeat-интервале НЕ перезаписывать весь объект. Вместо этого: прочитать текущий `status.json`, обновить ТОЛЬКО поле `heartbeat`, сохранить обратно. Не трогать `paused`, `running` и другие поля. Или ещё проще: heartbeat должен проверять `.ralph-pause` файл и устанавливать `paused` в соответствии с ним.
  - **Ограничения:** Не менять формат status.json.
  - **Критерии приёмки:** Нажать паузу в UI. Подождать 10 секунд (два цикла heartbeat). Проверить что status.json по-прежнему содержит `paused: true`.

- [ ] {{TASK:1.3}} Добавить лимит Content-Length на HTTP-запросы
  **ПОДРОБНОСТИ:**
  - **Что сделать:** В `ralph-tracker-web.py` метод `do_POST` читает `self.rfile.read(clen)` без проверки верхнего предела `clen`. Запрос с `Content-Length: 2000000000` вызовет `MemoryError`.
  - **Как сделать:** Добавить проверку `if clen > 1_000_000: self.send_error(413); return` перед `self.rfile.read(clen)`. 1MB — достаточно для любого API-запроса.
  - **Ограничения:** Не блокировать легитимные запросы (максимальный ожидаемый размер — несколько КБ для описания задачи).
  - **Критерии приёмки:** Сервер возвращает 413 при Content-Length > 1MB.

## Спринт 3: Надёжность и error handling
**[Role: Backend Architect]**

- [ ] {{TASK:3.1}} Заменить bare except pass на логирование ошибок
  **ПОДРОБНОСТИ:**
  - **Что сделать:** В `ralph-tracker-web.py` ~30 мест с `except: pass`. В `ralph-overseer.js` ~10 мест с `catch(e) {}`. Ошибки молча проглатываются, делая отладку невозможной.
  - **Как сделать:** Заменить каждый `except: pass` на `except Exception as e: print(f"[error] {context}: {e}")` где `context` — краткое описание операции (например, "save_busy_state", "parse_status_json"). В JS: `catch(e) { console.error('context:', e.message); }`. НЕ менять логику — только добавить логирование. Оставить `except: pass` только в `Tee.write()` и `Tee.flush()` (строки 44, 48) — там ошибка в логировании не должна ломать основной поток.
  - **Ограничения:** Не менять поведение программы. Только добавить вывод ошибок.
  - **Критерии приёмки:** Запустить сервер, намеренно повредить `projects.json` (невалидный JSON). Убедиться что ошибка выводится в консоль/лог, а не проглатывается.

- [ ] {{TASK:3.2}} Исправить socket leak в ralph-tray.py is_port_listening
  **ПОДРОБНОСТИ:**
  - **Что сделать:** В `ralph-tray.py` функция `is_port_listening()` создаёт socket, но если `connect()` бросает исключение, `close()` не вызывается.
  - **Как сделать:** Использовать `with`-контекст или `try/finally` для гарантированного закрытия сокета.
  - **Ограничения:** Нет.
  - **Критерии приёмки:** Функция корректно закрывает сокет в обоих путях (успех и ошибка).

- [ ] {{TASK:3.3}} Удалить мёртвый код после рефакторинга
  **ПОДРОБНОСТИ:**
  - **Что сделать:** Удалить: функцию `togglePanel` (пустая, deprecated), неиспользуемую переменную `cmd_str` (строка 679), CSS-класс `.card` (не используется в HTML), старые ссылки на `live_console_3.log`, `live_console_5.log` если есть.
  - **Как сделать:** Grep по файлу на каждое удаляемое имя, убедиться что нет живых ссылок, удалить.
  - **Ограничения:** Не удалять CSS-классы которые используются внутри динамически генерируемого HTML (renderSprintBlock и т.д.).
  - **Критерии приёмки:** `grep -c "togglePanel\|cmd_str\|\.card " ralph-tracker-web.py` возвращает 0.
