# Planning — SkyStrike

## 1. Архитектура проекта
- **Core Loop:** Паттерн Game Loop (Update -> Draw) с фиксированным `deltaTime`.
- **ECS (Entity Component System) Light:** Массив `entities`. Каждая сущность обязана иметь поле `type` и метод `update(dt)` и `draw(ctx)`.
- **Canvas Scaling:** Логическое разрешение 1080x1920. CSS масштабирует `canvas` с сохранением пропорций по центру экрана.

## 2. Структура директорий
```
/
├── index.html        # Точка входа, UI-оверлей
├── style.css         # Стили, центрирование Canvas (Letterboxing)
└── src/
    ├── main.js       # Инициализация игры, стейт меню
    ├── Engine.js     # Игровой цикл, управление сущностями, расчет dt
    ├── Player.js     # Логика корабля игрока, ввод
    ├── Enemies.js    # Классы врагов (Scout, Interceptor и т.д.)
    ├── Projectiles.js# Снаряды игрока и врагов
    ├── CollisionManager.js # Обработка столкновений на основе Матрицы
    ├── WaveDirector.js # Спавн врагов по времени
    └── ParticleSystem.js # Система визуальных эффектов (взрывы)
```

## 3. Матрица коллизий (Collision Matrix)
Обработка столкновений СТРОГО разделена по `type` сущности:
1. `player` <-> `enemy` (Урон обоим)
2. `player` <-> `enemy_projectile` (Урон игроку, снаряд уничтожен)
3. `player_projectile` <-> `enemy` (Урон врагу, снаряд уничтожен)
4. `player` <-> `powerup` (Игрок получает бафф, бонус уничтожается)

Запрещено проверять коллизии внутри одной группы (игрок с игроком, враг с врагом). Для фильтрации использовать строго проверку `entity.type === 'имя_типа'`, а не наличие свойства `hp`.
