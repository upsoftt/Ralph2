- [x] {{TASK:3.4}} Task Groups: Объединение задач в группы с подзадачами. [Role: Frontend Developer]
  ПОДРОБНОСТИ:
  **Что сделать:** Реализовать возможность объединять задачи в именованные группы (коллекции). Группа — это родительская задача, внутри которой находится список подзадач. Примеры: «Сбор в отпуск», «Сбор на природу», «Задачи по работе». Каждая подзадача внутри группы может отмечаться как выполненная независимо. Группа показывает прогресс выполнения (например, «3 из 5 выполнено»).
  **Как сделать:** Расширить модель данных в Store: добавить тип `TaskGroup` с полями `id`, `title`, `icon` (опционально, эмодзи), `subtasks: Task[]`, `createdAt`. В UI группа отображается как карточка-аккордеон: заголовок группы с иконкой свёрнуть/развернуть, прогресс-бар или счётчик выполненных подзадач, и раскрывающийся список подзадач внутри. Добавить кнопку «Создать группу» рядом с кнопкой добавления обычной задачи. При создании группы — модальное окно или инлайн-форма с полем названия и возможностью сразу добавить первые подзадачи. Внутри группы — собственное поле ввода для добавления новых подзадач. Подзадачи поддерживают toggle (выполнено/не выполнено) и удаление. Группы и их подзадачи сохраняются в localStorage через существующий механизм Store. Группы отображаются в общем списке наравне с обычными задачами и подчиняются фильтрации: фильтр «Done» показывает группы, где все подзадачи выполнены; «Active» — где есть невыполненные.
  **Ограничения:** Не использовать сторонние библиотеки для аккордеона или группировки. Вложенность только один уровень (группа → подзадачи, без вложенных групп). Группы должны корректно работать с существующим Drag & Drop (перетаскивание групп целиком, не подзадач между группами). Сохранять обратную совместимость данных в localStorage — старые задачи без групп должны продолжать работать.
  **Критерии приемки:** 1. Можно создать именованную группу задач. 2. Внутри группы можно добавлять, отмечать и удалять подзадачи. 3. Группа показывает прогресс выполнения (счётчик или прогресс-бар). 4. Группа сворачивается/разворачивается по клику на заголовок. 5. Фильтрация (All/Active/Done) корректно работает с группами. 6. Данные групп сохраняются в localStorage и восстанавливаются при перезагрузке. 7. Обратная совместимость с задачами без групп.
