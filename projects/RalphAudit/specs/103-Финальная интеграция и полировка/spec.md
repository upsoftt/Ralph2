- [x] {{TASK:103.3}} Добавить мини прогресс-бар выполнения подзадач под каждым спринтом
  **ПОДРОБНОСТИ:**
  - **Что сделать:** Под заголовком каждого спринта (`.master-header`) добавить тонкий зелёный прогресс-бар, аналогичный прогресс-бару проекта в левой панели. Бар показывает долю выполненных подзадач спринта. Без процентов, без текста — только зелёная полоска. Когда все подзадачи выполнены, спринт обводится зелёной рамкой (класс `.fully-completed` — уже реализован).
  - **Как сделать:**
    1. В функции `renderTaskStructure()` (строка ~1589) — после `<div class="master-header" ...>...</div>` вставить:
       ```html
       <div class="sprint-progress-bar"><div class="sprint-progress-fill"></div></div>
       ```
    2. CSS-стили:
       ```css
       .sprint-progress-bar {
           height: 3px;
           background: var(--border);
           width: 100%;
           flex-shrink: 0;
       }
       .sprint-progress-fill {
           height: 100%;
           background: var(--green);
           border-radius: 0 2px 2px 0;
           transition: width 0.3s ease;
           width: 0%;
       }
       .fully-completed .sprint-progress-fill {
           border-radius: 0;
       }
       ```
    3. В функции `updateTaskDataSurgically()` (строка ~1618) — после строки `mt.querySelector('.spec-counter').innerText = ...;` добавить обновление ширины прогресс-бара:
       ```javascript
       const progressFill = mt.querySelector('.sprint-progress-fill');
       if (progressFill) {
           const pct = det.total > 0 ? (det.completed / det.total * 100) : 0;
           progressFill.style.width = pct + '%';
       }
       ```
    4. Также в `renderTaskStructure()` сразу после вставки HTML установить начальную ширину из данных `det.completed / det.total`.
  - **Ограничения:**
    - Бар должен быть ровно между хедером и телом спринта (не внутри хедера и не внутри скрываемого тела).
    - Бар всегда виден, даже когда тело спринта свёрнуто (display:none).
    - Никаких процентов, текста или иконок — только зелёная полоска на сером фоне.
    - Не трогать существующую логику `.fully-completed` (зелёная рамка) — она остаётся как есть.
  - **Критерии приёмки:**
    - Под каждым спринтом видна тонкая полоска прогресса.
    - Полоска заполняется зелёным пропорционально выполненным подзадачам.
    - При 0 задач — полоска пустая (серая). При всех выполненных — полоска полностью зелёная + зелёная рамка спринта.
    - Плавная анимация при изменении прогресса (transition).
    - Прогресс обновляется при автоматическом обновлении данных (без перезагрузки страницы).

    3. Добавить `padding: 12px 20px;` для корректных отступов в sticky-режиме.
    4. Убедиться что `z-index` шапки выше, чем у `.split-resizer` (z-index: 10) и других элементов, чтобы шапка не перекрывалась при прокрутке.
    5. Учесть высоту шапки в расчёте `max-height` правой панели `.split-right` (текущее значение `calc(100vh - 40px)` может потребовать корректировки на высоту шапки).
    6. Проверить адаптивный режим (`@media max-width: 1200px`) — sticky должен работать и в мобильном layout.
  - **Ограничения:**
    - Не менять содержимое шапки (SVG-иконка, текст "Ralph Tracker 2.0").
    - Не ломать split-view layout — панели должны корректно располагаться под шапкой.
    - Использовать `position: sticky`, а не `position: fixed` — шапка остаётся в потоке документа и не требует дополнительных отступов.
    - Не менять `z-index` других элементов — только установить z-index шапки выше существующих.
  - **Критерии приёмки:**
    - При прокрутке страницы шапка остаётся вверху и не уходит за пределы экрана.
    - Контент не перекрывается шапкой (нет наложений).
    - Визуальное отделение шапки от контента (border-bottom) видно при прокрутке.
    - Split-view, горизонтальный и вертикальный ресайзеры работают корректно.
    - В адаптивном режиме (< 1200px) шапка также остаётся sticky.
