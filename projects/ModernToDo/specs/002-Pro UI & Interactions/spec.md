- [x] {{TASK:2.3}} Drag & Drop: Визуальное перетаскивание задач. [Role: Frontend Developer]
  ПОДРОБНОСТИ:
  **Что сделать:** Реализовать возможность визуального перетаскивания (drag & drop) элементов списка задач мышкой для изменения их порядка. Новый порядок должен сохраняться в Store и localStorage.
  **Как сделать:** Использовать нативный HTML5 Drag and Drop API (draggable, dragstart, dragover, drop). При начале перетаскивания элемент визуально «приподнимается» (тень + opacity). Показывать индикатор места вставки (линия или подсветка) между элементами. После drop — обновить массив задач в Store с новым порядком, сохранить в localStorage. Учесть работу с фильтрами: перетаскивание должно менять порядок в полном массиве, а не только среди отфильтрованных.
  **Ограничения:** Не использовать сторонние библиотеки (только нативный API). Drag & drop должен корректно работать в Chrome, Firefox, Edge. На мобильных устройствах (touch) достаточно fallback — без перетаскивания, порядок управляется иначе или отключён.
  **Критерии приемки:** 1. Задачи можно перетаскивать мышкой вверх/вниз. 2. Визуальная обратная связь при перетаскивании (тень, индикатор позиции). 3. Новый порядок сохраняется после перезагрузки страницы. 4. Работает корректно при активных фильтрах (All/Active/Done). 5. Плавная анимация перемещения элементов.

- [x] {{TASK:2.4}} Emoji Icon Picker: Назначение иконки-эмодзи для задачи. [Role: Frontend Developer]
  ПОДРОБНОСТИ:
  **Что сделать:** Дать пользователю возможность назначить любой эмодзи в качестве иконки для каждой задачи. Эмодзи отображается слева от текста задачи в списке. Если иконка не выбрана — показывается иконка по умолчанию (например, ○ или 📌).
  **Как сделать:** Добавить в модель задачи поле `icon` (строка, хранящая один эмодзи, по умолчанию пустое). При создании или редактировании задачи рядом с полем ввода отображать кнопку-триггер (текущий эмодзи или иконка «смайлик»). По клику на кнопку открывается компактный попап/дропдаун с сеткой популярных эмодзи (8–12 категорий: смайлы, объекты, еда, природа, и т.д.). Выбранный эмодзи сохраняется в Store и localStorage. Реализовать поиск внутри пикера по ключевым словам (например, «звезда» → ⭐). Для набора эмодзи использовать массив, встроенный в код (без сторонних библиотек).
  **Ограничения:** Не использовать сторонние npm-пакеты для Emoji Picker (реализовать самостоятельно). Попап не должен выходить за границы viewport. Пикер должен корректно закрываться при клике вне его области. Поддержка клавиатурной навигации внутри пикера (Tab, Enter, Escape для закрытия).
  **Критерии приемки:** 1. Каждая задача может иметь назначенный эмодзи, отображаемый слева от текста. 2. Попап-пикер открывается по клику и содержит категории эмодзи. 3. Работает поиск по эмодзи. 4. Выбранный эмодзи сохраняется в localStorage и восстанавливается при перезагрузке. 5. Пикер доступен с клавиатуры (Tab/Enter/Escape). 6. При отсутствии выбранного эмодзи отображается иконка по умолчанию.

- [x] {{TASK:2.5}} Background Image Picker: Назначение фоновой картинки для задачи. [Role: Frontend Developer]
  ПОДРОБНОСТИ:
  **Что сделать:** Дать пользователю возможность назначить фоновое изображение для каждой задачи (и группы задач). Картинка отображается как декоративный фон карточки задачи — полупрозрачно, чтобы текст оставался читаемым. Если картинка не задана — карточка отображается в стандартном стиле (стеклянный фон).
  **Как сделать:** Добавить в модель задачи поле `backgroundImage` (строка: data URL или пустое). В контекстном меню задачи (или по отдельной кнопке-иконке 🖼️) добавить опцию «Фоновая картинка». По клику открывается модальное окно с двумя способами выбора: (1) загрузка файла с устройства через `<input type="file" accept="image/*">` с превью, (2) выбор из встроенной коллекции пресетов (8–12 эстетичных паттернов/градиентов, хранящихся как CSS-градиенты или inline SVG). При загрузке пользовательской картинки: конвертировать в data URL через FileReader, ресайзить до max 400x300 через Canvas API для экономии localStorage. Фон карточки: `background-image` с `opacity: 0.15–0.25` через pseudo-element `::before`, чтобы не влиять на читаемость текста. Добавить кнопку «Убрать фон» для сброса к дефолту.
  **Ограничения:** Не использовать сторонние библиотеки. Максимальный размер изображения после сжатия — 50KB (data URL). При превышении — показать предупреждение и предложить более сильное сжатие. Учитывать лимит localStorage (~5MB): при переполнении — graceful fallback с уведомлением (интеграция с TASK 3.3). Фоновая картинка не должна мешать читаемости текста ни в тёмной, ни в светлой теме — обязательный затемняющий/осветляющий оверлей поверх картинки. Пресеты не занимают место в localStorage (CSS-градиенты). Модальное окно доступно с клавиатуры (Tab, Enter, Escape).
  **Критерии приемки:** 1. Для каждой задачи можно загрузить фоновую картинку с устройства. 2. Доступна галерея встроенных пресетов (градиенты/паттерны). 3. Картинка отображается полупрозрачным фоном карточки, текст остаётся читаемым. 4. Корректная работа в тёмной и светлой темах. 5. Сжатие изображения до ≤50KB перед сохранением. 6. Данные сохраняются в localStorage и восстанавливаются при перезагрузке. 7. Кнопка сброса фона к дефолтному стилю. 8. Модальное окно доступно с клавиатуры (Tab/Enter/Escape).
